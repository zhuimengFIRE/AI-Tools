1.项目规范

目录结构

AiDemo   	       # 项目名称
│   ├── Assets         # 资源
│   ├── Base 	       # 基类
│   ├── Info.plist     # 配置文件
│   ├── Modules        # 业务模块
│   │   ├── Add        # 添加
│   │   ├── Home       # 首页
│   │   └── Mine       # 个人中心
│   │       
│   └── SceneDelegate.swift
└── Pods

组织原则
- 保持项目结构清晰，遵循模块化原则
- 相关功能放在同一目录下
- 使用适当的目录命名，反映其功能


2.代码规范
命名规则

类名：PascalCase(大驼峰)
函数名：camelCase(小驼峰)
常量：UPPER_SNAKE_CASE(全大写，单词之间用下划线分割)
变量：camelCase(小驼峰)


代码质量原则
- 遵循 SOLID 设计原则
- 避免代码重复（DRY原则）
- 保持代码简洁、清晰、易读
- 考虑代码的可维护性和可拓展性
- 使用最新的Swift语言进行iOS开发

异常处理
- 合理使用异常处理机制
- 提供清晰的错误信息
- 记录必要的错误日志
- 优雅的处理边界情况


3.文档规范
- 为所有的类的方法添加代码注释，示例如下
/**
 * 函数功能说明
 * 
 * @Param 参数名 - 参数说明
 * @returns 返回值说明
 */
- 使用中文编写文档


4.测试规范
- 遵循测试金字塔原则
- 保持测试代码的整洁和可维护性
- 避免重复的测试代码
- 避免在代码中硬编码敏感信息
